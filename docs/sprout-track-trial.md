# sprout-track 試用ガイド

## 起動コマンド (SQLite版・最速)

```bash
docker run -d \
  --name sprout-track \
  --restart unless-stopped \
  -p 3000:3000 \
  -v sprout-track-db:/db \
  -v sprout-track-env:/app/env \
  -v sprout-track-files:/app/Files \
  sprouttrack/sprout-track:latest
```

起動後: http://localhost:3000

- **初期PIN:** `111222`
- **Family Manager:** `/family-manager` (パスワード: `admin`)

## 観察チェックリスト

### 1. オンボーディング体験
- [ ] 初回セットアップで詰まった箇所はどこか
- [ ] 「家族招待」までの導線の分かりやすさ
- [ ] 赤ちゃん情報の入力項目で足りない/多すぎる項目

### 2. 授乳記録のUX (最頻操作)
- [ ] **片手操作**できるか (赤ちゃんを抱っこしながら)
- [ ] 左右切替の操作回数
- [ ] タイマー開始から終了までのタップ数
- [ ] 夜間 (暗い画面) での視認性
- [ ] 誤タップしやすい箇所
- [ ] 開始時刻の事後修正のしやすさ

### 3. 両親リアルタイム共有 ★最重要
- [ ] 2デバイスで同じファミリーに入り、片方で授乳開始 → もう片方の画面に反映されるまでの時間
- [ ] 「今ママが授乳中です」のようなライブ状態表示の有無
- [ ] 同時編集時のコンフリクト挙動
- [ ] オフライン → 復帰時の同期挙動

### 4. 通知
- [ ] Web Push通知は届くか (iOS Safari でもOK?)
- [ ] 「前回授乳からX時間」リマインダーの精度・タイミング
- [ ] 通知音・バイブの設定

### 5. グラフ・統計
- [ ] 1日のリズム可視化が直感的か
- [ ] 週次・月次推移の見やすさ
- [ ] 「平均間隔」「左右バランス」等の有用指標

### 6. オフライン挙動
- [ ] 機内モードで記録 → 復帰時の同期
- [ ] 同期失敗時のリトライ・ユーザー通知
- [ ] データ欠損の発生有無

### 7. 多言語の不在による不便さ
- [ ] **日本語が無いことで、配偶者が使いにくいか**
- [ ] 専門用語 (Tummy Time等) の理解しにくさ
- [ ] 単位 (oz/lb) → 日本人として混乱するか

### 8. ぴよログ比較
ぴよログを使ったことがあれば、以下の観点で比較:
- [ ] 入力速度
- [ ] 一覧の見やすさ
- [ ] 共有のしやすさ
- [ ] グラフの質
- [ ] **何が劣っていて、何が優れているか**

### 9. 我々の差別化案の検証
- [ ] **「次は誰の番」機能の必要性** — 実生活で困るか?
- [ ] **左右別の母乳量推定** — あったら使うか?
- [ ] **進行中授乳の家族間表示** — どれくらい価値があるか?

### 10. 「結局自作する価値があるか」判定
試用後、以下の問いに答える:
- [ ] sprout-track のままで足りる? → **自作中止**を検討
- [ ] 多言語(ja/vi)だけ足りない? → **fork** (ライセンス的に不可だが学習用) または別実装
- [ ] UXに根本的不満? → **独自実装の根拠あり**
- [ ] リアルタイム共有が弱い? → **独自実装の差別化ポイント**
